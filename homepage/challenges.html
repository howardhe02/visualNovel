Challenges

Possible challenges Part 1

We plan on implementing an inventory system that will be intractable in all screens. The challenge here is that it must be accessible in all parts of the game and transfer data between all phases of the game. The inventory will be sorted alphabetically so there will be a use of bubble sorting, so the player can easily find items.

Possible challenges Part 2

In addition to an inventory system, we plan to create a game where the player will be able to do as much as they want. The game will provide as much freedom to the player as possible meaning that functions and variables must be tailored as vaguely as possible but still achieve accurate outputs based on the input the player gives.
